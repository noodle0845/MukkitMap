# 먹킷맵 UX/UI 개선 제안서

> 목표: 트렌디한 SaaS 대시보드 톤 안에서, 사용자가 “고민 없이 바로” 장소를 등록·확인할 수 있게 만든다.

---

## 1. 현재 문제점 요약

### 1-1. 프로젝트 상세 페이지 (가장 큰 병목)

좌측 사이드바에 5개 패널이 항상 동시에 펼쳐져 있다.

```
[참여자 목록]   ← 닉네임당 삭제 버튼
[참여자 추가]   ← 입력 3개 + 저장 버튼
[네이버 장소 검색] ← 검색창 + 결과별 "장소로 입력" 버튼
[장소 추가 폼] ← 입력 7개 + 저장 버튼
[필터]         ← select 3개 + 초기화 버튼
```

오른쪽엔 지도 + 상세 카드 + 장소 리스트가 또 쌓인다. 카드 한 장마다 외부링크/수정/삭제 아이콘 버튼 3개가 노출된다.

→ **한 화면 평균 노출 버튼 ≈ 11~14개**, 메인 CTA로 보이는 emerald 버튼만 5개 이상.
→ 사용자는 “어디서 시작해야 하지?”에서 멈춘다.

### 1-2. 흐름 단절

`네이버 검색 → 결과 클릭 → 폼 자동 채움 → 저장`이 위·아래 패널로 끊겨 있다. 사용자는 한 번 등록할 때마다 시선을 4번 이동시킨다.

### 1-3. 정보 우선순위 평탄

모든 패널이 동일한 `panel + panel-header + section-title` 컴포넌트를 쓴다. 시각적 비중이 똑같아 어디가 “지금 해야 할 액션”인지 알기 어렵다.

### 1-4. 디자인 톤

Arial 시스템 폰트, 빨강·노랑·파랑·초록 pill이 동시 노출, 그림자/라운드가 단조로움. 관리자 툴 느낌은 강하지만 “현대적 웹 서비스” 톤은 약하다.

### 1-5. 피드백

- 삭제 시 `window.confirm()` → 디자인 일관성 깨짐.
- 저장/수정 후 별도 토스트 없음.
- 로딩 상태가 “네이버 지도 불러오는 중” 텍스트뿐.

### 1-6. 홈 화면

빈 사용자(프로젝트 0개)와 헤비 유저(프로젝트 多)가 같은 무거운 히어로 + 우측 320px 폼을 본다. 폼은 “필요할 때만” 모달로 띄우는 게 맞다.

## 2. UX 개선 전략

| 원칙 | Before | After |
|---|---|---|
| 첫 인지 부하 | 폼 + 리스트 + 히어로 동시 | 빈/채워진 상태 분리, 폼은 모달 |
| 화면당 메인 CTA | 5개 이상 | **1개** (FAB 또는 헤더 1개) |
| 등록 흐름 | 검색 / 폼이 다른 패널 | 한 슬라이드 패널의 [검색 → 폼] 한 흐름 |
| 보조 액션 | 카드마다 3개 아이콘 노출 | `⋯` 더보기 메뉴 |
| 필터 | select 3개 + 초기화 | 참여자 dot row + 카테고리 chip row |
| 삭제 확인 | 시스템 `confirm()` | 인라인 토스트 + 실행 취소 5초 |
| 참여자 관리 | 사이드바 항상 노출 | “참여자 관리” 슬라이드 패널 |

핵심 원칙 4가지:

1. **하나의 화면, 하나의 핵심 액션** — 부가 기능은 다 숨긴다.
2. **목적지부터 보여주기** — 지도가 메인이다. 폼은 필요할 때만.
3. **흐름은 한 패널 안에서 끝낸다** — 검색·입력·저장이 같은 슬라이드 패널 안.
4. **상태는 색이 아니라 위계로** — 한 색만 강조, 나머지는 무채색.

## 3. UI 디자인 방향

### 3-1. 컬러 (3색만 사용)

| 토큰 | 값 | 용도 |
|---|---|---|
| `--primary` | `#10b981` (emerald 500) | 메인 CTA, 활성 상태 |
| `--primary-soft` | `#ecfdf5` | 활성 칩 배경, 강조 영역 |
| `--accent` | `#f97316` | 새 항목, 알림 등 미세 강조 (1곳만) |
| `--bg` | `#f8fafc` | 페이지 배경 |
| `--surface` | `#ffffff` | 카드/패널 |
| `--border` | `#e2e8f0` | 구분선 |
| `--text` / `--text-muted` / `--text-soft` | `#0f172a` / `#64748b` / `#94a3b8` | 위계 |
| 상태 | `success #22c55e`, `warning #f59e0b`, `danger #ef4444` | 토스트·에러만 |

기존 빨강/노랑/파랑/초록 pill 4종 동시 노출은 제거. 강조는 항상 emerald 한 색.

### 3-2. 타이포

- **Pretendard Variable** (한글/숫자 SaaS 표준)을 CDN에서 로드.
- 위계: `display 28/700` → `title 18/700` → `body 14/500` → `caption 12/600 (uppercase tracking)`.
- 숫자는 `font-feature-settings: "tnum"`로 표 정렬.

### 3-3. 형태

- 라운드: 입력 12px, 버튼 12px, 카드 16px, 시트 24px.
- 그림자 3단계: `sm` 호버, `md` 카드, `lg` 떠 있는 시트/팝오버.
- 여백 8단위(8/12/16/20/24/32). 카드 내부 20–24px, 섹션 사이 24px.

### 3-4. 인터랙션

- hover: `translateY(-1px) + shadow-md`
- focus: `ring-2 ring-emerald-200`
- 버튼 active: `scale(0.98)`
- 시트/모달: 220ms `cubic-bezier(.2,.8,.2,1)` slide-in
- 토스트: 우하단, 4초, 실행 취소 가능

### 3-5. 컴포넌트 어휘

`btn-primary`, `btn-ghost`, `btn-icon`, `chip`, `chip-active`, `card`, `sheet`, `kebab`, `toast`, `field`, `field-row`. 기존 `panel/section-title/pill`과 호환되도록 alias 유지.

## 4. 화면 흐름 재설계

### 4-1. 첫 방문 흐름 (목표: 첫 장소 등록까지 5클릭)

```
홈(빈 상태) ─[새 프로젝트]→ 모달(이름/설명) ─[생성]→ 프로젝트 페이지
   1클릭          2클릭                 3클릭

→ 우상단 [+ 장소 추가] ─→ 슬라이드 패널 [검색 탭]
   4클릭                     검색어 입력 → 결과 클릭
                                                5클릭 [저장]
```

기존 흐름은 9클릭(참여자 추가까지 별도)이었음. **5클릭으로 단축**.

### 4-2. 재방문 사용자 흐름

```
홈 ─→ 프로젝트 카드 클릭 ─→ 지도 + 리스트 즉시 노출
                              ↓
                          [+] FAB 1개로 등록
                          [⋯] 카드별 수정/삭제
                          [필터 칩] 1탭으로 원하는 친구만
```

### 4-3. 참여자 관리 흐름 (덜 자주 쓰는 기능)

```
헤더 "👥 참여자 4" 칩 클릭 → 슬라이드 패널
   - 추가: 닉네임 + 색상만 (역할은 기본값)
   - 삭제: 카드 hover 시 ⋯ 더보기 안에
```

상시 노출 → 필요 시 호출로 전환.

## 5. 버튼/기능 구조 개선안

### 5-1. 화면당 노출 버튼 수

| 화면 | Before | After | 감소율 |
|---|---|---|---|
| 홈 (빈 상태) | 7개 | 2개 | -71% |
| 홈 (프로젝트 多) | 8+N개 | 1+N개 (카드 클릭) | -60%↑ |
| 프로젝트 상세 | 11~14개 | 4개 + FAB | -65% |
| 장소 카드 (개별) | 4개(카드+3아이콘) | 1개(카드 전체) | -75% |

### 5-2. 화면에 남길 / 숨길 기능

**프로젝트 상세 — 항상 노출 (4개)**

1. 헤더 좌측: 뒤로가기 (`<-`)
2. 헤더 우측: 공유 링크 (`Share`)
3. 필터 바: 참여자 dot 칩 + 카테고리 칩
4. 우하단 FAB: `+ 장소 추가`

**숨김 (호출형)**

| 기능 | 어디로 | 호출 방식 |
|---|---|---|
| 참여자 추가/삭제 | Sheet | 헤더 “참여자 4명” 칩 클릭 |
| 장소 검색·등록 | Sheet | FAB 클릭 |
| 장소 수정 | Sheet | 카드 ⋯ → 수정 |
| 장소 삭제 | 토스트(실행 취소) | 카드 ⋯ → 삭제 |
| 외부 링크 | Sheet 안 상세 | 카드 클릭 시 상세 패널 |
| 필터 초기화 | 칩 “전체” 자동 | 별도 버튼 제거 |
| 샘플 데이터 | 빈 홈에만 | 빈 상태에서 ghost 버튼 1개 |

### 5-3. 버튼 네이밍

| Before | After |
|---|---|
| 네이버 지도 열기 | 지도 보기 |
| 한 줄 후기 | 메모 |
| 마커 색상 | 색상 |
| 필터 초기화 | (제거, “전체” 칩으로 자동) |
| 공유 링크 복사 | 공유 |
| 이 주소로 장소 입력 | 이 위치로 등록 |
| 샘플 데이터 만들기 | 예시 보기 |

## 6. 추천 레이아웃 구조

### 6-1. 프로젝트 상세 (Desktop ≥1280px)

```
┌─────────────────────────────────────────────────────────────────┐
│ ←  부산 친구 맛집지도                       👥 4명   [공유]      │  ← Header
│    친구들이 추천한 부산 맛집…                                     │
├─────────────────────────────────────────────────────────────────┤
│  ◉전체  ◯라면  ◯워렌  ◯존슨  ◯베일      🍽 밥집 ☕ 카페 …    │  ← Filter bar
├──────────────────────────────────┬──────────────────────────────┤
│                                  │  📍 장소 4곳                │
│                                  │ ┌──────────────────────┐ ⋯ │
│              🗺 지도                │ │ ◯ 라면 · 카페          │  │
│                                  │ │ 전포 감성카페           │ │
│       (마커 hover → 미니 카드)     │ │ 부산 부산진구 전포동      │ │
│                                  │ └──────────────────────┘   │
│                                  │ ┌──────────────────────┐ ⋯ │
│                                  │ │ ◯ 워렌 · 밥집          │  │
│                                  │ └──────────────────────┘   │
│                                  │                              │
│                                  │                  [+ 장소]   │  ← FAB
└──────────────────────────────────┴──────────────────────────────┘
```

- **좌 60% 지도, 우 40% 리스트**, 모바일은 1단(지도 위, 리스트 아래) + FAB.
- 상세 카드는 슬라이드 패널 (우측 480px), 모바일은 bottom sheet.
- 필터 바는 sticky, 칩만 사용.

### 6-2. 홈 (Desktop)

빈 상태:
```
┌────────────────────────────────────────────────────────────┐
│   먹킷맵                                                    │
│   카톡방에 흩어진 친구 맛집을 한 지도에 모으세요.            │
│                                                            │
│   [+ 새 프로젝트]   [예시 보기]                              │
```

```
├─────────────────────────────────────────────────┤
│ ┌──────────┐ ┌──────────┐ ┌──────────┐         │
│ │ 부산 친구  │ │ 회사 점심  │ │ 도쿄 출장  │   …    │
│ │ 4명·12곳   │ │ 6명·8곳    │ │ 2명·3곳    │         │
```

## 7. 적용 후 기대 효과

| 지표 | Before | After |
|---|---|---|
| 화면당 평균 노출 버튼 | ~11개 | ~4개 |
| 화면당 메인 CTA | 5개 | 1개 |
| 첫 장소 등록까지 클릭 | 9 | 5 |
| 카드당 액션 노출 | 4개 | 1개(+⋯) |
| 색상 팔레트 동시 노출 | 4색 | 1색(+상태) |
| 폰트 시스템 | Arial | Pretendard |
| 피드백 일관성 | confirm 대화상자 혼용 | 토스트 통일 |

정성적으로:

- 첫 사용자: 무엇을 눌러야 할지 즉시 파악.
- 헤비 사용자: 등록 한 번에 화면 이동 1회 이내.
- 디자인 인상: “관리자 도구” → “현대적 웹 서비스”.
