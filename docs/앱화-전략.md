# 먹킷맵 앱화 전략

먹킷맵은 기존 웹앱 기능을 유지하면서 다음 순서로 앱처럼 확장한다.

1. PWA: 홈화면에 추가해서 앱처럼 실행
2. Android: TWA 또는 Capacitor로 패키징
3. iOS: Capacitor 기반 패키징 검토

## 현재 적용한 PWA 구조

- `public/manifest.json`으로 앱 이름, 아이콘, 테마 색상, standalone 실행 방식을 정의한다.
- `public/sw.js`로 PWA 설치 조건을 만족하되, 로그인/초대/프로젝트/지도 데이터 요청은 캐시하지 않는다.
- `src/components/PwaInstallPrompt.tsx`에서 Android 설치 프롬프트, iOS 홈 화면 추가 안내, PC 안내를 분기 처리한다.
- iOS Safari용 `apple-touch-icon`, 상태바 메타 태그, 홈화면 제목을 `src/app/layout.tsx`에 추가했다.

## Android 패키징 검토

### TWA

장점:
- 기존 Vercel 웹앱을 거의 그대로 유지할 수 있다.
- 로그인, 초대 링크, 지도 기능이 웹과 동일하게 동작한다.
- 앱 업데이트 대부분을 웹 배포로 처리할 수 있다.
- PWA 구조와 가장 자연스럽게 연결된다.

단점:
- 앱 내부가 사실상 Chrome 기반 웹 실행 환경이다.
- 오프라인/네이티브 기능 확장에는 한계가 있다.
- Digital Asset Links 설정과 Play Store 서명 설정이 필요하다.

### Capacitor

장점:
- Android/iOS를 같은 방식으로 관리할 수 있다.
- 추후 푸시 알림, 네이티브 공유, 파일, 위치 권한 같은 기능을 붙이기 쉽다.
- 상태바, 스플래시, 앱 아이콘을 더 세밀하게 제어할 수 있다.

단점:
- 네이티브 프로젝트 관리가 추가된다.
- 웹 배포와 앱 배포 버전 관리가 따로 생긴다.
- 단순 웹앱 유지 목적이라면 TWA보다 작업량이 많다.

### 먹킷맵 추천

Android는 우선 TWA가 더 적합하다.

이유:
- 현재 핵심 기능이 전부 웹 기반이다.
- 지도, 로그인, 초대 링크는 웹 URL 흐름이 중요하다.
- Vercel 배포만으로 기능 업데이트를 계속 반영할 수 있다.
- PWA 다음 단계로 가장 자연스럽다.

Capacitor는 푸시 알림, 네이티브 위치 권한, 연락처 초대, 네이티브 공유 시트 같은 기능을 본격적으로 넣을 때 전환을 검토한다.

## iOS 패키징 검토

iOS App Store까지 고려하면 Capacitor 기반 패키징이 현실적이다.

검토 항목:
- Safari PWA 홈화면 추가로 먼저 사용성을 검증한다.
- App Store 제출 시 단순 웹뷰 앱으로 보이지 않도록 앱 고유 경험을 보강한다.
- 카카오/구글 로그인 리디렉션 URL과 Supabase Auth 설정을 iOS 앱 스킴까지 확장한다.
- 위치 권한, 지도, 개인정보 처리방침, 이용약관을 심사 기준에 맞게 정리한다.

## 스토어 등록 전 체크리스트

- 앱 아이콘: 1024px 원본, Android adaptive icon, iOS icon set
- 스플래시: 밝은 배경, 먹킷맵 로고 중심
- 앱 이름: 먹킷맵
- 영문 이름: Mukkit Map
- 개인정보 처리방침 URL
- 이용약관 URL
- 로그인 제공자 설정: Google, Kakao, Supabase callback URL
- 초대 링크 딥링크: `/invite/:inviteCode`
- 위치 정보 사용 여부 명시
- 지도 제공자 약관 확인
- 테스트 계정 또는 심사용 안내 준비
- 삭제/탈퇴/데이터 삭제 정책 정리

## 개인정보와 권한

현재 필요한 정보:
- 로그인 식별자
- 닉네임
- 프로필 이미지
- 프로젝트/멤버/장소 데이터
- 사용자가 직접 입력한 주소와 좌표

추후 위치 기반 기능을 추가할 때 필요한 항목:
- 위치 권한 요청 목적
- 현재 위치를 서버에 저장하는지 여부
- 위치 기반 주변 장소 검색의 처리 방식

권장 문구:
- "먹킷맵은 친구들과 공유할 맛집 지도를 만들기 위해 로그인 식별자, 닉네임, 프로필 이미지, 프로젝트 참여 정보, 사용자가 입력한 장소 정보를 저장합니다."
- "현재 위치 기능을 사용하는 경우 주변 장소 검색과 지도 중심 이동에만 활용하며, 별도 저장 여부는 기능 구현 시 명확히 안내합니다."
